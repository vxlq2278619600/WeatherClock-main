## 软件架构
本项目基于 `STM32F407 + FreeRTOS + ST7789 + ESP AT + AHT20` 实现，当前代码结构如下：

- `app/`
  - 应用主流程、页面逻辑、天气解析、UI 消息分发、工作队列
- `driver/`
  - 屏幕、RTC、AHT20、ESP AT、按键、LED、串口等外设驱动
- `firmware/`
  - `CMSIS` 和 `STM32F4xx Standard Peripheral Library`
- `third_lib/`
  - `FreeRTOS`
- `resources/`
  - 界面设计图和图片资源
- `mdk/`
  - Keil 工程文件与编译输出

## 硬件接线说明

### ST7789 SPI 屏幕接线

| 屏幕引脚 | STM32F407 核心板 |
| ---- | ---- |
| GND  | GND  |
| VCC  | 3V3  |
| SCL  | PB13 |
| SDA  | PC3  |
| RES  | PE3  |
| DC   | PE4  |
| CS   | PE2  |
| BLK  | 3V3  |

### 引脚说明
- GND：屏幕地线，需要与 STM32F407 核心板共地。
- VCC：屏幕供电，当前接 3.3V。
- SCL：SPI 时钟线，对应 STM32F407 的 `PB13`。这里的 `SCL` 是 SPI `SCK`，不是 I2C `SCL`。
- SDA：SPI 数据线，对应 STM32F407 的 `PC3`。这里的 `SDA` 是 SPI `MOSI`，不是 I2C `SDA`。
- RES：屏幕复位引脚，对应 `PE3`。
- DC：命令/数据选择引脚，对应 `PE4`，用于区分当前发送的是命令还是显示数据。
- CS：SPI 片选引脚，对应 `PE2`，用于选中屏幕进行通信。
- BLK：背光控制引脚。当前固定接 `3V3`，使背光常亮。

### 调试结论
1. 当前项目代码使用 `PB13` 作为 SPI 时钟线，使用 `PC3` 作为 SPI 数据线。
2. 当前 ST7789 SPI 屏幕已经通过基础显示测试。
3. 屏幕自检测试已正常显示黑、白、红、绿、蓝等颜色页面，并能正常显示英文和数字。
4. 说明 SPI 通信、屏幕初始化、`RES/DC/CS` 控制线和基本显示方向已经验证通过。
5. `BLK` 当前接 `3V3`，而不是接 MCU 控制引脚。这样可以让背光保持常亮，便于后续排查显示内容问题。
6. 需要注意：核心板原理图上的板载 TFT 接口默认引脚可能与当前项目代码使用的引脚不完全一致。当前外接屏幕应以本 README 中的接线表和项目代码为准。

### 常见注意事项
- 背光亮不代表屏幕通信成功。只有能正常显示颜色、文字或图形，才说明 SPI 显示链路基本正常。
- `SCL` 和 `SDA` 名称容易误解，它们在本屏幕中属于 SPI 接口，不是 I2C 接口。
- 如果屏幕只有背光没有内容，优先检查 `SCL`、`SDA`、`RES`、`DC`、`CS` 是否接错。
- 如果画面颜色异常，可能需要检查 `RGB565` 颜色格式、屏幕初始化参数或字节发送顺序。
- 如果画面方向不对，后续可以通过 ST7789 的显示方向参数或 `MADCTL` 配置调整。
- 后续修改代码前，建议保留当前“屏幕自检正常”的版本作为稳定回退点。

## 当前已实现功能

### 1. 系统主流程
- 上电后完成板级初始化
- 启动 FreeRTOS 调度器
- 支持欢迎页、WiFi 连接页、主页等页面流程
- 已具备工作队列和周期任务框架

### 2. 图形界面
- 已实现欢迎页
- 已实现 WiFi 连接页
- 已实现主页面布局
- 已实现错误页
- UI 使用独立任务和消息队列串行刷新，避免多个任务直接抢占屏幕

### 3. 时间功能
- 支持 SNTP 网络校时
- 支持 RTC 本地时间保存与读取
- 主页面可刷新时间与日期

### 4. 传感器与环境数据
- 已接入 AHT20 驱动
- 支持采集室内温度
- 支持采集室内湿度
- 已完成“AHT20 -> 串口 -> ST7789 屏幕”的本地显示测试

### 5. 网络与天气
- 已接入 ESP AT 固件通信
- 支持连接指定 WiFi
- 支持查询当前 WiFi 连接状态
- 支持 HTTP GET 请求
- 已接入心知天气实时天气接口
- 已支持解析城市、天气现象、天气代码、室外温度

## 当前未完成或未接入功能

### 1. 页面整合仍待完善
- AHT20 本地显示测试已完成，但还没有完全整理进最终主页布局
- 天气、自定义页面、自检页面之间的切换策略仍需整理

### 2. 配网方式仍是硬编码
- WiFi 名称和密码目前仍写在源码中
- 还没有配网页、按键输入、串口配置或手机配网功能

### 3. 断网与异常恢复能力不足
- WiFi 初始化失败时会直接进入错误页
- WiFi 连接失败时会直接进入错误页
- 当前没有自动重连、失败重试、回退逻辑

### 4. 驱动具备但业务层未完全使用
- `key` 按键驱动已存在，但未接入完整页面交互
- `led` 驱动已存在，但未用于状态指示
- `bl24c512` 驱动已存在，但未用于参数保存
- 串口接收回调能力已存在，但未实现命令交互

### 5. 产品化功能尚未实现
- 没有设置页
- 没有城市切换
- 没有多日天气预报
- 没有闹钟/提醒功能
- 没有用户参数持久化保存
