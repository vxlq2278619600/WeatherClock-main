# WeatherClock
梅花嵌入式天气时钟项目源码

## Bilibili教学视频
[【全开源手把手项目实战】基于FreeRTOS的stm32智能天气时钟](https://www.bilibili.com/video/BV1tfL1zeEQN)

## 软件架构
本项目基于 `STM32F407 + FreeRTOS + ST7789 + ESP AT + AHT20` 实现，当前代码结构大致如下：

- `app/`
  - 应用主流程、页面逻辑、天气解析、UI 消息分发、工作队列
- `driver/`
  - 屏幕、RTC、AHT20、ESP AT、按键、LED、串口等外设驱动
- `firmware/`
  - `CMSIS` 和 `STM32F4xx Standard Peripheral Library`
- `third_lib/`
  - `FreeRTOS`
- `resources/`
  - 界面设计图和图片资源
- `mdk/`
  - Keil 工程文件与编译输出

## 当前已实现功能

### 1. 系统主流程
- 上电后完成板级初始化
- 启动 FreeRTOS 调度器
- 依次显示欢迎页、WiFi 连接页、主页
- 创建周期性任务，持续更新时间、室内环境和联网天气

### 2. 图形界面
- 已实现欢迎页
- 已实现 WiFi 连接页
- 已实现主页面布局
- 已实现错误页
- UI 使用独立任务和消息队列进行串行刷新，避免多个任务直接抢占屏幕

### 3. 时间功能
- 支持 SNTP 网络校时
- 支持 RTC 本地时间保存与读取
- 主页面每秒刷新一次时间与日期
- 时间显示带闪烁冒号效果

### 4. 传感器与环境数据
- 已接入 AHT20
- 支持采集室内温度
- 支持采集室内湿度
- 主页可动态刷新室内温湿度

### 5. 网络与天气
- 已接入 ESP AT 固件通信
- 支持连接指定 WiFi
- 支持查询当前 WiFi 连接状态
- 支持 HTTP GET 请求
- 已接入心知天气实时天气接口
- 已支持解析城市、天气现象、天气代码、室外温度
- 主页已支持室外温度和天气图标刷新

## 当前未完成或未接入功能

### 1. 天气显示仍有缺口
- 虽然已经解析出城市名和天气文字，但主页目前没有动态显示这两项
- 当前主页中的城市名仍是初始化时写死的默认值

### 2. 配网方式仍是硬编码
- WiFi 名称和密码目前写在源码中
- 还没有配网页、按键输入、串口配置或手机配网功能

### 3. 断网与异常恢复能力不足
- WiFi 初始化失败时会直接进入错误页
- WiFi 连接失败时会直接进入错误页
- 当前没有自动重连、失败重试、回退逻辑

### 4. 驱动具备但业务层未使用
- `key` 按键驱动已存在，但未接入页面交互
- `led` 驱动已存在，但未用于状态指示
- `bl24c512` 驱动已存在，但未用于参数保存
- 串口接收回调能力已存在，但未实现命令交互

### 5. 产品化功能尚未实现
- 没有设置页
- 没有城市切换
- 没有多日天气预报
- 没有闹钟/提醒功能
- 没有用户参数持久化保存

## 推荐下一步开发顺序

1. 补全主页上的室外城市和天气文字动态显示
2. 增加 WiFi 断线检测与自动重连
3. 增加按键交互和设置页
4. 增加 EEPROM 参数保存，去掉 WiFi/城市硬编码
5. 再扩展多日天气、闹钟提醒等功能

## 当前项目定位
从目前代码状态看，这个项目已经完成了“天气时钟主干功能”：

- 能开机
- 能连网
- 能校时
- 能读取室内温湿度
- 能获取实时天气
- 能在彩屏主页上动态显示主要信息

但它还没有完全进入“可配置、可交互、可长期稳定运行”的成品阶段，当前更适合作为一个完整的嵌入式综合练手项目或二次开发基础工程。
